Docker简介

Docker是一个开源的应用容器引擎，开发者可以使用其快速搭建轻量的虚拟环境，打包应用及依赖，实现Linux环境下的快速部署和移植。
Docker的优势主要体现在持续集成性、隔离性、安全性。
Docker通过镜像文件来分发，通过镜像仓库解决了分发问题

Docker安装和环境配置

作为深度炼丹爱好者，强烈建议使用nvidia-docker以解决cuda、cudnn、显卡适配等一系列烦恼。正所谓前人种树，后人乘凉，本着不重复造轮子的初衷，下文以nvidia-docker的安装和配置作示例讲解整个流程。

  1. 卸载Docker旧版本
sudo apt-get remove docker docker-engine docker.io
  1. 安装系统工具
sudo apt-get update
sudo apt-get -y install apt-transport-https ca-certificates curl software-properties-common
  1. 安装GPG证书

在国内建议使用国科大源：

curl -fsSL https://mirrors.ustc.edu.cn/docker-ce/linux/ubuntu/gpg | sudo apt-key add -

若是安装官方源则如下所示：

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
  1. 写入软件源
sudo add-apt-repository \
    "deb [arch=amd64] https://mirrors.ustc.edu.cn/docker-ce/linux/ubuntu \
    $(lsb_release -cs) \
    stable"
  1. 更新并安装Docker-CE

拉取合适的Docker

为了更便捷地搭建深度学习环境，可直接到[NVIDIA docker官网]针对自己所需Ubuntu、cuda和cudnn版本拉取配置好的镜像。例docker pull nvidia/cuda:10.2-cudnn7-runtime-ubuntu16.04。

base: starting from CUDA 9.0, contains the bare minimum (libcudart) to deploy a pre-built CUDA application.
Use this image if you want to manually select which CUDA packages you want to install.

runtime: extends the base image by adding all the shared libraries from the CUDA toolkit.
Use this image if you have a pre-built application using multiple CUDA libraries.

devel: extends the runtime image by adding the compiler toolchain, the debugging tools, the headers and the static libraries.
Use this image to compile a CUDA application from sources.

宿主机cuda安装

以最基本的方法，分开安装驱动和cuda
宿主机的cuda和docker的cuda需对应 选择runfile格式
https://developer.nvidia.com/cuda-toolkit-archive
以10.2cuda为例，
wget https://developer.download.nvidia.com/compute/cuda/10.2/Prod/local_installers/cuda_10.2.89_440.33.01_linux.run
sudo sh cuda_10.2.89_440.33.01_linux.run

配置Anaconda环境

可通过[清华源Anaconda安装包合集]下载，例wget https://mirrors.tuna.tsinghua.edu.cn/anaconda/archive/Anaconda3-5.3.1-Linux-x86_64.sh。
下载后使用bash Anaconda3-5.3.1-Linux-x86_64.sh执行安装，一路Enter+yes即可(除了最后VSCode的请求视要求no)。
使用以下命令检查conda安装情况
source ~/.bashrc
conda --version

Docker基本语句

保存容器为镜像
容器一旦退出，所有内容不复存在！
使用docker commit {container_ID} {new_name}语句保存，可通过命令行开头的当前容器ID，或docker ps查询当前执行的容器ID。例docker commit 51539ee1281e lizexian/gpu_base:cuda_10.2_cudnn_7_anaconda3，名字中冒号后的部分为tag内容。
