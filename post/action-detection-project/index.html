虚拟环境

项目配置在Anaconda虚拟环境fight中，使用conda activate fight启动
虚拟环境核心依赖版本：Python=3.7.9 Pytorch=1.5.0 torchvision=0.6.0

项目路径

项目核心目录在/home/lenovo/fight下
待检测视频存放在/home/lenovo/fight/slowfast/fight_video下
检测结果存放在/home/lenovo/fight/slowfast/result下

执行方法

执行demo时只需cd到/home/lenovo/fight/slowfast路径下，执行语句python tools/run_net.py --cfg /home/lenovo/fight/slowfast/demo/AVA/fight.yaml即可。

打架检测格式

会将打架的人标注侯选框及置信度，每2s执行一次检测，打架行为标注为fight/hit(a person)

基于任务的yaml文件参数修改

针对不同任务，对/home/lenovo/fight/slowfast/demo/AVA/fight.yaml文件参数进行修改，只需更改yaml文件最下DEMO部分的三个参数WEBCAM，INPUT_VIDEO和OUTPUT_FILE，当然，所有任务都需要谨慎填写输出的绝对路径，默认mp4格式
本地存放的几个视频较长(可能大于1h)，根据需要可提前结束：用双ctrl+c退出，不影响视频保存；mkv格式适合直接运行程序，mp4格式适合摄像头拍摄，因为读取该类数据会拉爆显存

  1. 本地视频动作检测
    注释掉WEBCAM语句，使用INPUT_VIDEO语句，填好对应视频的绝对路径
  2. 摄像头对应场景动作检测
    注释掉INPUT_VIDEO语句，使用WEBCAM语句，WEBCAM对应编号填写ls /dev/video*获得的编号，默认为2，如果不正确就从0至3再尝试一下
    注：摄像头拍摄的画面受限于镜头清晰度、光照、抖动等因素，效果较本地视频有明显差距，但在实际应用场景中(高精度固定摄像头)可达到本地动作检测的精度，此仅作演示

备注：为了代码运行，会将TRAIN/TEST的ENABLE参数置为False，NUM_GPUS置为1，今后只在此基础上改动即可。

conda环境配置

基本环境为Anaconda5.3.1 + cuda10.2 + cudnn7
此处记录铁科院服务器环境配置 重点在于Detectron2在CUDA10.1下的安装，cuda10.1与pytorch1.6的适配性和cuda10.1与显卡驱动的适配性

conda create -n fight python=3.7.9
conda install pytorch==1.6.0 torchvision==0.7.0 cudatoolkit=10.1 -c pytorch
conda install git
pip install 'git+https://github.com/facebookresearch/fvcore'
pip install simplejson
conda install av -c conda-forge
conda install -c iopath iopath
pip install psutil
pip install opencv-python

pip install -U cython
pip install -U 'git+https://github.com/facebookresearch/fvcore.git' 'git+https://github.com/cocodataset/cocoapi.git#subdirectory=PythonAPI'
git clone https://github.com/facebookresearch/detectron2 detectron2_repo
python -m pip install detectron2 -f https://dl.fbaipublicfiles.com/detectron2/wheels/cu101/torch1.6/index.html

git clone https://github.com/facebookresearch/slowfast
# add to path
export PYTHONPATH=/path/to/slowfast/slowfast:$PYTHONPATH

git clone https://github.com/facebookresearch/slowfast
cd slowFast
python setup.py build develop
